## 阶段2：认证 + 权限

**目标**：实现登录和权限控制

**change 名称**：`auth-and-permission`

**依赖**：阶段1

**Entity**：
- [ ] sys_user (管理员)
- [ ] sys_role (角色)
- [ ] sys_permission (权限)
- [ ] sys_user_role (用户-角色)
- [ ] sys_role_permission (角色-权限)

**后端任务**：
- [ ] AuthModule：登录、JWT 生成、刷新令牌
- [ ] JwtGuard：JWT 验证守卫
- [ ] PermissionGuard：权限验证守卫
- [ ] @CurrentUser 装饰器
- [ ] @RequirePermission 装饰器
- [ ] 初始化种子数据（管理员、角色、权限）

**前端任务**：
- [ ] 登录页面
- [ ] 登录状态管理
- [ ] 路由权限控制
- [ ] 菜单动态渲染
- [ ] Token 刷新逻辑

**验收标准**：
- 能使用 admin/123456 登录
- 登录后能看到菜单
- 无权限访问返回 403
- Token 过期能自动刷新
