## 一、全新项目第一步：先定边界

你要先搞清楚 4 件事

### 业务范围

1、业务范围做什么？核心流程是什么？
后台管理 + C 端小程序 / H5？有没有支付、订单、推送？
2、量级与并发初期
用户多少？峰值多少？需不需要高可用？中等项目 → 不用上微服务，单体足够。
3、工期与交付
快速上线 = 精简架构

## 二、第二步：选定技术栈（全栈架构师核心决策）

全新项目，你要一次性拍死技术栈，避免后期重构。

### 前端

管理后台：React（选你熟的）
移动端：H5 / Taro
构建：umi4
请求封装：axios + 拦截器
组件库：antd
样式：tailwindcss

### 后端（你现在的方向）

框架：NestJS
语言：TypeScript
ORM：TypeORM
数据库：MySQL
缓存：Redis
消息队列：Kafka/RabbitMQ（中等项目建议使用）
鉴权：JWT + 刷新令牌
部署：Docker + Nginx

## 第三步：设计数据库

全新项目最关键的就是表结构一次设计到位。
梳理所有实体
1、设计关系：一对多、多对多
2、必须字段：id、create_time、update_time、is_deleted
3、提前建索引：外键、查询字段、排序字段
4、权限系统直接上 RBAC 5 张表
5、中等项目表数量一般在 20～40 张 之间。

## 第四步：搭建项目骨架

### 整体架构采用 monorepo 搭建

- 后端骨架（Nest）
  2、项目结构模块化（User、Auth、Role、Permission、System 等）
  3、统一返回格式
  4、全局异常过滤器
  5、JWT 鉴权守卫
  6、权限守卫（基于 RBAC）
  7、请求参数校验（DTO）
  8、TypeORM 实体全部建好
  9、日志、跨域、安全配置、限流
  10、数据库连接、Redis 连接
  11、环境配置（dev/test/prod）

- 前端骨架
  1、路由结构
  2、登录、Layout
  3、请求封装（统一携带 token、错误提示）
  4、权限
  5、菜单动态生成
  6、工具类、公共组件

## 第五步：制定开发规范（避免项目后期烂尾）

你必须定死规则，强制执行：
1、接口统一 RESTful
2、禁止在 Controller 写业务
3、禁止循环查库（N+1）
4、禁止在 service 里写复杂 SQL
5、所有查询必须走 Repository
6、事务必须加，且范围不能过大
7、敏感接口做幂等、防重
8、密码加密、数据脱敏
9、日志必须可追踪
10、规范不是约束，是降低维护成本。

## 第六步：核心业务流程设计

### 优先做

1、用户体系
2、登录、注册、JWT、刷新 token
3、权限体系
4、角色、菜单、按钮权限、接口鉴权
5、核心主流程
比如：
商品 → 订单 → 支付 或 后台审核流程

## 通用能力

1、文件上传、导入导出、字典、配置、消息推送
2、能异步的全部扔MQ：日志、站内信、推送、统计、后续履约动作
